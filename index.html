Introduction

What is an API?

API stands for Application Programming Interface. It is a set of rules and protocols that allows two software applications to communicate with each other.

Think of an API as a contract between two systems. It defines what requests can be made, how to make them, and what responses to expect — without revealing the internal implementation.

🍽️

The Restaurant Analogy

You (client) sit at a table. The menu (API documentation) shows what's available. You give your order to the waiter (API). The waiter takes it to the kitchen (server). The kitchen prepares the food and the waiter brings it back. You never enter the kitchen — that's the abstraction!

🔌

The Power Socket Analogy

A power socket is an API. You don't need to know how electricity is generated — you just plug in your device. The socket defines the interface (shape, voltage), and anything that fits can use it.

Importance

Why APIs are Important?

APIs are the backbone of modern software development. Here's why every developer needs to understand them.

Modularity & Reusability

Build once, use everywhere. A payment API like Razorpay can be integrated into thousands of apps without rebuilding payment logic each time.

Security & Abstraction

APIs hide sensitive internal logic. External systems interact only through defined endpoints, protecting your database and business logic.

Speed of Development

Use Google Maps API instead of building mapping from scratch. Developers save months of work by consuming existing APIs.

Scalability

Microservices communicate via APIs. Each service can scale independently, making large applications more manageable and resilient.

Integration & Ecosystem

Connect Slack with GitHub, Jira with Trello, or CRM with ERP. APIs make third-party integrations seamless and powerful.

Platform Independence

A single REST API can serve web, Android, iOS, and IoT devices simultaneously without any code changes on the backend.

API Types

Types of APIs

APIs come in many flavors — understanding each type helps you choose the right tool for the right job.

Most Popular

REST API

REST (Representational State Transfer) is an architectural style for building APIs over HTTP. It's stateless, uses standard HTTP methods, and returns data in JSON or XML.

  • Uses HTTP methods: GET, POST, PUT, DELETE, PATCH
  • Stateless — each request is independent
  • Resources identified by URLs
  • Returns JSON/XML responses
  • Used by: Twitter, GitHub, Stripe, Google
REST endpoints
# CRUD operations on /users resource GET /api/users # List all users GET /api/users/1 # Get user by ID POST /api/users # Create new user PUT /api/users/1 # Update user fully PATCH /api/users/1 # Update partially DELETE /api/users/1 # Delete user # Response format { "id": 1, "name": "Ram Prajapati", "role": "Data Analyst", "status": 200 }
Facebook / Meta

GraphQL API

GraphQL is a query language for APIs that allows clients to request exactly the data they need. Created by Facebook in 2015, it solves the over-fetching and under-fetching problems of REST.

  • Single endpoint — typically /graphql
  • Client specifies exact fields needed
  • No over-fetching or under-fetching
  • Strongly typed schema
  • Used by: GitHub, Shopify, Facebook
GraphQL query
# GraphQL Query — get only what you need query { user(id: "1") { name email posts { title createdAt } } } # Response — exactly what was asked { "data": { "user": { "name": "Ram Sir", "email": "ram@technoglobe.in", "posts": [{ "title": "API Guide" }] } } }
Enterprise Legacy

SOAP API

SOAP (Simple Object Access Protocol) is a protocol that uses XML for messaging. It's highly standardized, secure, and used in enterprise and banking applications.

  • Uses XML exclusively
  • Strict standards — WSDL defines contract
  • Built-in security (WS-Security)
  • Works over HTTP, SMTP, TCP
  • Used by: Banks, PayPal (legacy), SAP
SOAP envelope
<!-- SOAP Request (XML) --> <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org" xmlns:usr="http://user.service.com"> <soapenv:Header/> <soapenv:Body> <usr:GetUser> <usr:UserId>42</usr:UserId> </usr:GetUser> </soapenv:Body> </soapenv:Envelope>
Google — High Performance

gRPC API

gRPC (Google Remote Procedure Call) uses Protocol Buffers (Protobuf) for serialization and HTTP/2 for transport. It's extremely fast and used in microservices.

  • Uses Protobuf — binary, smaller than JSON
  • HTTP/2 — multiplexing, streaming
  • Strongly typed contracts (.proto files)
  • 5–10x faster than REST for internal services
  • Used by: Netflix, Square, Google Cloud
user.proto
// Protocol Buffer definition (.proto) syntax = "proto3"; message UserRequest { string user_id = 1; } message UserResponse { string name = 1; string email = 2; int32 age = 3; } service UserService { rpc GetUser(UserRequest) returns (UserResponse); }
Real-Time Communication

WebSocket API

WebSocket provides full-duplex, persistent connections between client and server. Unlike HTTP, the connection stays open so data flows both ways in real time.

  • Bidirectional — server can push to client
  • Persistent connection (no reconnect overhead)
  • Low latency — ideal for live data
  • Uses ws:// or wss:// protocol
  • Used by: WhatsApp, Slack, stock tickers
websocket_client.js
// JavaScript WebSocket Client const ws = new WebSocket( 'wss://api.example.com/live' ); ws.onopen = () => { ws.send(JSON.stringify({ type: 'subscribe', channel: 'stock_prices' })); }; ws.onmessage = (event) => { const data = JSON.parse(event.data); console.log(`Price: ₹${data.price}`); };
Python Framework

Flask API

Flask is a lightweight Python web framework used to build REST APIs quickly. It's minimal, flexible, and perfect for beginners and small to medium projects.

  • Micro-framework — minimal boilerplate
  • Easy routing with @app.route decorators
  • JSON responses with jsonify()
  • Supports Flask-RESTful extension
  • Great for learning and prototyping
flask_api.py
from flask import Flask, jsonify, request app = Flask(__name__) users = [ {"id":1, "name":"Ram Sir"}, {"id":2, "name":"Alice"} ] @app.route('/api/users', methods=['GET']) def get_users(): return jsonify(users) @app.route('/api/users/<int:uid>', methods=['GET']) def get_user(uid): user = [u for u in users if u["id"]==uid] return jsonify(user[0]) if user else ("Not Found",404) @app.route('/api/users', methods=['POST']) def create_user(): data = request.get_json() users.append(data) return jsonify(data), 201 if __name__ == '__main__': app.run(debug=True)
Modern Python — Fast & Async

FastAPI

FastAPI is a modern, high-performance Python framework built on Pydantic and Starlette. It's async-native, auto-generates docs, and is 2–3x faster than Flask.

  • Auto-generates Swagger UI docs
  • Async/await support natively
  • Type hints with Pydantic validation
  • One of the fastest Python frameworks
  • Used by: Microsoft, Netflix, Uber
fastapi_app.py
from fastapi import FastAPI, HTTPException from pydantic import BaseModel app = FastAPI(title="User API") class User(BaseModel): name: str email: str age: int @app.get("/users/{user_id}") async def get_user(user_id: int): return {"id": user_id, "name": "Ram Sir"} @app.post("/users", status_code=201) async def create_user(user: User): # Pydantic auto-validates input! return {"message": "Created", "data": user} # Run: uvicorn fastapi_app:app --reload # Docs at: http://localhost:8000/docs
API Specification Standard

OpenAPI / Swagger

OpenAPI (formerly Swagger) is a specification for describing REST APIs. It defines the structure of your API in a YAML or JSON file, enabling auto-generated documentation and client SDKs.

  • YAML/JSON-based API specification
  • Swagger UI generates interactive docs
  • Auto-generate client SDKs in any language
  • Industry standard for API documentation
  • Supported by FastAPI, Flask-RESTX, etc.
openapi.yaml
openapi: "3.0.0" info: title: User API version: "1.0.0" paths: /users: get: summary: Get all users responses: "200": description: OK content: application/json: schema: type: array items: $ref: '#/components/schemas/User'

Flask — Full CRUD API
Flask
from flask import Flask, jsonify, request from datetime import datetime app = Flask(__name__) students = [] next_id = 1 # GET all students @app.route('/students', methods=['GET']) def get_all(): return jsonify({"count": len(students), "data": students}) # POST — create student @app.route('/students', methods=['POST']) def create(): global next_id body = request.get_json() student = { "id": next_id, "name": body["name"], "course": body.get("course", "Data Analytics"), "joined": datetime.now().isoformat() } students.append(student) next_id += 1 return jsonify(student), 201 # DELETE student @app.route('/students/<int:sid>', methods=['DELETE']) def delete(sid): global students students = [s for s in students if s["id"] != sid] return jsonify({"message": "Deleted"}) if __name__ == '__main__': app.run(debug=True, port=5000)
